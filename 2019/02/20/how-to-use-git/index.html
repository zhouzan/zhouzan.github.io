关于HEAD指针

git关于版本控制的命令中，经常会出现HEAD，而HEAD类似于指针，指向当前的提交（commit），在实际使用中，HEAD等价于当前提交的哈希，在知道哈希的情况下，可以用哈希值替换HEAD，但是HEAD使用更方便，不需要了解哈希值即可。并且，HEAD可以用于访问相对提交记录，比如”倒数第二次提交“等这种形式。

回滚代码

回滚代码有一下四种情形：修改之后尚未add、已经add但尚未commit、已经commit但尚未push、已经push到了远端。

1. 修改之后尚未add，使用checkout命令

2. 已经add但尚未commit，使用reset命令

–soft 仅移动HEAD，回退commit版本，不恢复文件，不修改文件的add状态
–mixed 移动HEAD，不恢复文件，但是恢复文件的add状态，即add的文件变成未add状态
–hard 移动HEAD，还恢复文件，把已经修改的文件恢复成原始的样子，丢弃所有的修改

3. 已经commit但尚未push，使用reset命令

4. 已经push到了远端

分两种方法：保留远端的提交记录、删除远端的提交记录

a. 保留远端记录，提交“逆操作”，使用revert命令

注意：一般一次revert操作仅针对一次的commit，而不会一次revert多次commit

注意：revert操作可能会产生冲突

b. 删除远端记录，使用reset命令
